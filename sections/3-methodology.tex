\chapter{Methodology}
\label{chap:method}

This chapter walks through the Stage~1 method as it is actually implemented in my code, and then describes the Stage~2 generative design that sits on top of it. The order roughly follows the data path: I start with the problem setup and the variational autoencoder objective (\S\ref{sec:method_setup}), then the data pipeline including the Hilbert serialisation (\S\ref{sec:method_data}), the encoder with both its global and local variants (\S\ref{sec:method_encoder}), the reconstruction objective (\S\ref{sec:method_recon}), the decoder including the token-local variant and the anchor-relative position decoding (\S\ref{sec:method_decoder}), the small-scale semantic experiments that motivated several design choices (\S\ref{sec:method_semantic}), and finally the Stage~2 generative model (\S\ref{sec:method_stage2}). I close with an explicit summary of the seven configurations forming the large-scale ablation (\S\ref{sec:method_configs}).

The base architecture follows Can3Tok \citep{gao2025can3tok}: a Perceiver-style cross-attention encoder lifting a set of Gaussian primitives into a fixed token sequence, a KL bottleneck, and a transformer-plus-MLP decoder producing back the per-primitive parameters. Everything I add modifies this skeleton, but the encoder topology, the latent dimensionality, the decoder transformer block, and the loss family all come from Can3Tok.

\section{Framework and Problem Setup}
\label{sec:method_setup}

\paragraph{Scene representation.} A scene $\cG = \{g_i\}_{i=1}^{N}$ is a set of $N$ Gaussian primitives. Each primitive is a 14-dimensional vector
\begin{equation}
g_i = (\vmu_i,\; c_i,\; \alpha_i,\; s_i,\; q_i) \in \bR^3 \times \bR^3 \times (0,1) \times \bR^3_+ \times \bS^3,
\label{eq:prim}
\end{equation}
with the indices laid out positions [0:3], colour [3:6], opacity [6:7], scale [7:10], quaternion [10:14]. The covariance is $\Sigma_i = R(q_i)\,\mathrm{diag}(s_i^2)\,R(q_i)^\top$.

I work with a fixed budget of $N = 40{,}000$ Gaussians per scene, selected from the larger pre-fitted scene by top-$N$ opacity ranking (the Can3Tok default).

\paragraph{Autoencoder objective.} Stage~1 is a VAE \citep{kingma2014vae} with encoder $E_\phi$ and decoder $D_\theta$. The encoder produces a token-structured latent $Z \in \bR^{T \times d}$ with $T = 512$ tokens and $d = 32$ per-token dimensions, drawn from a diagonal Gaussian posterior $q_\phi(Z \mid \cG)$. The decoder reconstructs the Gaussian set. The training objective is the standard ELBO,
\begin{equation}
\cL = \cL_{\text{recon}}(\hat{\cG}, \cG) \;+\; \beta\,D_{\mathrm{KL}}\bigl(q_\phi(Z \mid \cG)\;\|\;\mathcal{N}(0, I)\bigr),
\label{eq:vae}
\end{equation}
with $\beta = 10^{-5}$. The very small $\beta$ keeps the latent informative for reconstruction; downstream generation tolerates this because the flow-matching Stage~2 model can warp arbitrary tractable noise into the latent distribution, regardless of how non-Gaussian the latter is.

\section{Data Pipeline}
\label{sec:method_data}

\subsection{Dataset}
\label{sec:method_dataset}

I train on SceneSplat-7K \citep{li2025scenesplat}, a collection of 7{,}916 indoor 3DGS scenes reconstructed from a mix of ScanNet \citep{dai2017scannet}, ScanNet++, ARKitScenes, Matterport3D, Replica, and similar indoor sources. The dataset ships with per-Gaussian ScanNet-72 category labels, which I use for the semantic experiments in \S\ref{sec:method_semantic}.

The main training set in the large-scale runs is 3{,}800 chunked indoor scenes preprocessed at $8\times8$ tile stride with $6\times6$ overlap from the ScanNet/Matterport sources. Three additional sources are folded in for the actual runs: 800 full indoor scenes, 1{,}290 ARKitScenes, and 906 ScanNet++ v2 scenes. Per-Gaussian semantic labels are disabled for the additional sources to avoid cross-dataset taxonomy mismatch. The total training pool is about 6{,}796 scenes. Validation uses 100 held-out full scenes (the primary thesis-target metric) and 88 disjoint held-out chunks (an in-distribution diagnostic).

Each scene is normalised into a canonical sphere of radius 10\,m. For chunked scenes, a global coordinate frame is precomputed from the union of all chunks belonging to the same parent scene and stored in a \texttt{norm\_factor.npy} file, so every chunk shares one consistent frame. Full scenes are normalised per scene. This normalisation is the precondition for any kind of cross-scene canonical ordering.

\subsection{Hilbert Serialisation}
\label{sec:method_hilbert}

After the per-scene opacity-based sampling of $N = 40{,}000$ Gaussians, I serialise the selected primitives along a 3D Hilbert curve \citep{skilling2004hilbert}. Each Gaussian's position is mapped to an integer grid in $[0, 2^{10})^3$ using a fixed canonical frame of radius 10\,m (the same radius as the normalisation), encoded with the Skilling AxesToTranspose construction, and finally sorted by the resulting scalar key $h(i)$. The flag in the code is called \texttt{morton\_order}, but the actual curve is selected by \texttt{order\_curve} and is \texttt{hilbert} in every run reported.

Two design points are critical here. First, the locality property of the Hilbert curve: consecutive indices are always spatially adjacent in the underlying grid, with no long jumps. The alternative Z-order (Morton) curve fails this property because a high-order bit flip moves the curve to a distant cell, which is the precise reason \citet{wu2024ptv3} prefer the Hilbert order for point cloud processing. Second, the frame is \emph{fixed}, not per-scene. With a per-scene frame the curve would traverse the same relative grid in every scene, but the same absolute slot index would correspond to a different physical region across scenes, defeating the purpose of the ordering for the slot-aligned loss. With a fixed canonical frame, slot $i$ corresponds to roughly the same physical region across all scenes, which makes the reconstruction target identifiable in the way RQ1 requires.

After sorting, contiguous blocks of $g = \lceil N/T \rceil = \lceil 40000/512 \rceil = 79$ Gaussians correspond to compact local clusters. This partition is reused in two places downstream: the per-token Hilbert-block anchors used by the position scaffold (\S\ref{sec:method_anchor}), and the per-token decoding done by the token-local decoder (\S\ref{sec:method_decoder}). Token $k$ owns block $k$ throughout.

\subsection{Colour Decomposition}
\label{sec:method_colour}

Colour is decomposed into a per-scene mean and a per-primitive residual. The mean colour $\bar{c} \in \bR^3$ is predicted from the scene-level shape embedding (or, in the disentangled variants, from $z_s$) by a small \texttt{MeanColorHead}, and the decoder produces a residual $r_i \in \bR^3$ for each primitive. The reconstructed colour is then
\begin{equation}
\hat{c}_i = \hat{\bar{c}} + \hat{r}_i.
\label{eq:colour_residual}
\end{equation}
This DC/AC split (an analogy to DC and AC coefficients in spectral decompositions) is on in every large-scale run. Its purpose is to remove the easy global colour-palette signal from the per-primitive prediction so the decoder can use its capacity on the harder local colour variation. In practice the per-scene mean is well predicted from the global summary, and the residual head only has to model local deviations from that mean.

\section{Encoder}
\label{sec:method_encoder}

The encoder topology comes from Can3Tok \citep{gao2025can3tok} and the underlying Perceiver design \citep{jaegle2021perceiver, jaegle2022perceiverio}. I describe the global form first, then the local variant that is one of the central architectural changes of the thesis.

\paragraph{Input features.} Each of the $N$ Gaussians produces an 18-dimensional feature vector at the input: voxel-centre coordinates from an FNV hash of a $40^3$ grid (3 dims), a voxel id (1 dim), the normalised position (3 dims), the colour (3 dims), opacity (1 dim), scale (3 dims), and quaternion (4 dims). The position and voxel centre are each lifted to 8-frequency Fourier features \citep{tancik2020fourier}, and the resulting vector is projected to the model width $w = 384$ by a linear input layer.

\paragraph{Global cross-attention bottleneck.} The Perceiver block uses a set of $1 + K$ learnable query tokens that cross-attend to the projected per-Gaussian features. Query 0 produces the scene-level shape embedding $e \in \bR^w$; the remaining $K$ tokens produce per-region geometry tokens. The token count $K$ is set by the YAML config (the value used in this work yields $T = 512$ latent tokens after the KL projection). Six self-attention layers then mix the query tokens. The output is a $(1 + K) \times w$ token sequence that is permutation-invariant in the input Gaussians.

\paragraph{KL bottleneck.} A linear projection per token produces the per-token mean and log-variance, the latent is sampled by reparameterisation, and an inverse linear projection brings it back to width $w$ for the decoder. The pre-KL flat embed dimension and the per-token dimension are constrained so that the total number of latent numbers is $T \cdot d = 512 \cdot 32 = 16384$. This dimensionality is held fixed across all seven large-scale variants; only its \emph{structure} changes, not its budget.

\paragraph{Spatially local encoder.} The single biggest change to the encoder is a windowed local variant. Each of the $K = 512$ geometry queries attends only to the Gaussians belonging to a local Hilbert window of $\pm \omega$ blocks around its own block, with $\omega = 1$ in my runs (so a query sees three contiguous Hilbert blocks). A single global CLS query still attends to all $N$ Gaussians, so the scene-level shape embedding remains a global summary. After the cross-attention, the encoder runs no global self-attention; cross-token mixing is handled later in the decoder transformer.

Two consequences follow. First, every latent token is the encoding of a local spatial cluster, not a global mixture of the whole scene. Two scenes with a similar local patch (a flat floor, a uniform wall) activate the same local features. This is the compositional structure I show in Chapter~\ref{chap:results} to be the key to closing the train-to-test generalisation gap. Second, when paired with structured\_latent (which uses the per-token posterior directly, instead of flattening and remixing it through a single dense projection), the latent now has the property latent token $k$ $=$ encoder block $k$ $=$ input Hilbert block $k$, end to end. The mapping is exact, not approximate.

\section{Reconstruction Objective}
\label{sec:method_recon}

The reconstruction term in Eq.~\eqref{eq:vae} is an element-wise $\ell_2$ residual on the 14-dimensional Gaussian vector. With the Hilbert order in place and the per-token block partition fixed, the slot-aligned loss has a stable target.

In the no-scaffold variants the predicted and target are compared directly:
\begin{equation}
\cL_{\text{recon}} \;=\; \frac{1}{B}\sum_{i=1}^{N} \bigl\lVert \hat{g}_i - g_i \bigr\rVert_2,
\label{eq:recon_noscaffold}
\end{equation}
with the sum implicitly broken into the per-attribute components (position, colour, opacity, scale, rotation) for reporting. A colour weight of 2.0 and a separate scalar weight on the mean-colour-prediction MSE are used in every large-scale run.

In the scaffolded variants, position is decoded relative to a per-token anchor (\S\ref{sec:method_anchor}), so the position part of the loss is computed on the offset $\hat{o}_i = \hat{\vmu}_i - a_{\tau(i)}$ rather than on the absolute coordinate. An additional anchor loss supervises the per-token anchor to be the centroid of its Hilbert block,
\begin{equation}
\cL_{\text{anchor}} \;=\; \frac{1}{T}\sum_{k=1}^{T} \bigl\lVert \hat{a}_k - \bar{\vmu}_{\text{block}(k)} \bigr\rVert_2^2,
\label{eq:anchor_loss}
\end{equation}
where $\bar{\vmu}_{\text{block}(k)}$ is the mean position of the Gaussians assigned to block $k$ by the Hilbert order. This is what makes the anchor head a meaningful localiser and not just a pass-through.

\section{Decoder}
\label{sec:method_decoder}

The decoder is a 12-layer transformer at width $w = 384$ followed by a per-primitive head that produces the 14-dimensional output. I describe the per-primitive head in two variants below, and then the anchor-relative position decoding.

\subsection{Flat vs.\ Token-Local Decoder}
\label{sec:method_tld}

The Can3Tok decoder ends with a single flat MLP. Concretely it concatenates all $T \cdot w = 512 \cdot 384 = 196{,}608$ post-transformer features into one vector, projects through a 1024-dimensional hidden bottleneck, and then projects to $N \cdot 14 = 560{,}000$ outputs. The parameter count is dominated by these two linear layers and reaches about 777M parameters at $N = 40{,}000$.

The 1024-dimensional bottleneck is the choke point. Every scene-specific local detail has to flow through it. With a few hundred scenes the network can almost memorise the relevant codes, but with thousands of scenes that memorisation strategy breaks and the model produces a generic, washed-out scene. The bottleneck cannot store the per-scene context needed to disambiguate similar input patches across the training set.

The token-local decoder replaces the flat MLP with a shared per-token MLP applied independently to each of the $T = 512$ decoder tokens. Each token decodes its own $g = 79$ Gaussians:
\begin{equation}
\hat{g}_{[k]} \;=\; \mathrm{MLP}_\theta\bigl(h_k\bigr), \quad k = 1, \ldots, T,
\label{eq:tld}
\end{equation}
where $h_k \in \bR^w$ is the post-transformer feature of token $k$ and $\hat{g}_{[k]} \in \bR^{g \times 14}$ is the predicted block. The MLP weights are shared across all tokens. The total parameter count drops from about 777M to about 1.6M, while the effective representation bandwidth per token rises from $1024/T \approx 2$ to $w = 384$. There is no longer a cross-scene bottleneck on the per-Gaussian path.

Token-local decoding gives a clean spatial structure: token $k$ attends to its local context through the (windowed) encoder, the latent slot $k$ is the per-token posterior of that context, the transformer block $k$ mixes it with its neighbours, the position scaffold places token $k$ at the centroid of its Hilbert block, and the per-token MLP decodes the $g$ Gaussians within. Each step matches the others. This end-to-end consistency is what makes the architecture generalise.

\subsection{Anchor-Relative Position Decoding}
\label{sec:method_anchor}

In the baseline decoder, the position head predicts the absolute coordinate of every Gaussian. Even with a Hilbert-ordered input, this still requires the network to learn an absolute coordinate target across scenes. Inspired by Scaffold-GS \citep{lu2024scaffold} and the structured prediction in GaussianCube \citep{zhang2024gaussiancube}, I split the position prediction into an anchor and a bounded local offset,
\begin{equation}
\hat{\vmu}_i \;=\; a_{\tau(i)} \;+\; \rho \,\tanh(o_i),
\label{eq:anchor_relative}
\end{equation}
where $a_{\tau(i)} \in \bR^3$ is the anchor of the token owning Gaussian $i$, $o_i \in \bR^3$ is a raw offset predicted by the per-Gaussian decoder, and $\rho > 0$ is a learnable global scale (exp-parameterised, initialised at 2.0). The anchors are predicted by a small head from the post-transformer token features and supervised by Eq.~\eqref{eq:anchor_loss} towards the Hilbert-block centroids.

Two practical points are easy to miss in the implementation but matter for downstream Stage~2:
\begin{itemize}
\item \textbf{Hilbert-block assignment.} The token that owns Gaussian $i$ is determined by the Hilbert order, not by voxel containment. Concretely, $\tau(i) = \lfloor i/g \rfloor$ with $g = \lceil N/T \rceil = 79$. This is reproducible at inference time from the index alone, with no need for the ground-truth position; it is the property that makes the scaffold Stage-2 feasible.
\item \textbf{Bounded local offset.} The $\tanh$ bounding is what makes the anchor genuinely informative. Without it, the offset is unbounded and the network can simply offset away from a meaningless anchor, falling back to absolute position prediction. With the $\tanh$, the local-region structure becomes the dominant signal.
\end{itemize}

\section{Disentanglement and the Two Latent Schemas}
\label{sec:method_disentangle}

Two of the seven Stage~1 variants additionally split the latent into a global layout component and a local geometry component, motivated by the small-scale results in \S\ref{sec:method_semantic} and detailed in the local-disentangle code path.

In these variants, a separate global layout latent $z_s \in \bR^{16 \times 32}$ is produced by a small projection of the encoder CLS into mean and log-variance vectors; this $z_s$ is KL-regularised on its own. The geometry latent is the structured per-token $z_g \in \bR^{512 \times 32}$ coming from the local encoder. The total per-scene latent has $16 + 512 = 528$ tokens but the geometry budget is unchanged.

The decoder consumes $z_g$ as its main sequence and accepts $z_s$ as a separate cross-attention context. Two structural losses help bind the meaning of the split:
\begin{itemize}
\item \textbf{Cross-reconstruction (cross\_recon).} For a batch of size $B$, I swap $z_s$ between scenes and re-decode with the original $z_g$. The reconstruction is graded against the original scene's geometry, so geometry must stay fixed under the swap. This forces $z_s$ not to carry geometry.
\item \textbf{Orthogonality.} A small batch-level penalty that decorrelates random subsets of the dimensions of $\mu_s$ and $\mu_g$, discouraging trivial duplication between the two subspaces.
\end{itemize}
Three small auxiliary heads on $z_s$ predict the scene's mean colour, ScanNet-72 label distribution, and per-category spatial centroids, all of which are unambiguously layout-level quantities.

For the rest of the document I call the latent schema of the seven variants in two groups:
\begin{itemize}
\item \emph{Flat schema} (variants 1--5): a single 512-token latent. The decoder consumes it directly; there is no separate $z_s$.
\item \emph{Split schema} (variants 6 and 7): a 16-token $z_s$ plus a 512-token $z_g$.
\end{itemize}
Stage~2 has to react to this difference.

\section{Small-Scale Semantic Experiments}
\label{sec:method_semantic}

Before committing to a Stage~1 design at scale, I ran a series of 300-scene experiments to understand which latent-structure choices were even feasible and which would survive the move to thousands of scenes. The motivation for the disentanglement step in \S\ref{sec:method_disentangle} comes directly from these.

\paragraph{Latent disentanglement variants.} Three configurations were tested.
\begin{description}
\item[Strategy~C (baseline)] No disentanglement. The latent is a single undifferentiated 512-token sequence.
\item[Strategy~A] A disentangled split where $z_s$ is derived from the shape embedding via a separate mean/variance projection and $z_g$ comes from the geometry queries. Auxiliary heads on $z_s$ predict mean colour, label distribution, and per-category centroids. Cross-reconstruction and orthogonality are active.
\item[Strategy~B1] A different conditioning route: a deterministic \texttt{Layout16Projector} produces a fixed 16-token layout code from the shape embedding (no VAE sampling on $z_s$). The latent has 512 pure geometry tokens, and the layout code conditions every decoder layer through cross-attention. Because the layout code is deterministic, the same partial scene always produces the same layout, which makes B1 ideal for the completion task.
\end{description}
A fourth variant, an additive-bias version of B (Strategy~B2), was also tried; it failed to disentangle (the additive bias is absorbed by the decoder as a constant and provides no useful conditioning). I report it briefly in Chapter~\ref{chap:results} for completeness.

\paragraph{Per-Gaussian InfoNCE.} On top of the disentanglement variants, a projection head maps each decoded Gaussian to a 32-d normalised embedding $f_i$. Following \citet{oord2018cpc}, the loss pulls together Gaussians that share a ScanNet-72 label and pushes apart Gaussians from different categories,
\begin{equation}
\cL_{\text{NCE}} \;=\; -\frac{1}{M}\sum_{i=1}^{M}\log \frac{\exp(f_i^\top p_{y_i}/\tau)}{\sum_{k=1}^{K_{\text{cats}}} \exp(f_i^\top p_k/\tau)},
\label{eq:infonce}
\end{equation}
where $p_k$ is the L2-normalised mean feature of category $k$ over the current batch, $y_i$ is the Gaussian's ScanNet-72 label, $M$ is the number of labelled Gaussians in the batch, and $\tau$ is a temperature ($\tau = 0.10$ in the small-scale runs, $\tau = 0.07$ for the variants that condition on $z_s$). Three flavours were tested: pgNCE on per-Gaussian decoder features, PoolNCE on pooled $z_s$ features, and Pool+pgNCE combining both. Of these, Pool+pgNCE gave the cleanest cross-scene swap behaviour. None of the three changes the reconstruction loss materially; the supervision is a soft secondary signal.

\paragraph{What carried over to the large-scale runs.} The 300-scene results (Chapter~\ref{chap:results}, \S\ref{sec:results_300}) confirmed that a disentangled latent with auxiliary heads is feasible, that B1's deterministic layout code is the right structure for partial-scan completion, and that per-Gaussian semantic features cluster meaningfully. The large-scale runs include two configurations with this disentanglement plus per-Gaussian InfoNCE (variants 6 and 7); five baseline-style variants without it complete the ablation. Strategy~B2 was not carried over.

\section{Stage~2 Generative Model}
\label{sec:method_stage2}

The second stage models the distribution of the frozen Stage~1 latent with flow matching \citep{lipman2023flowmatching, liu2023rectified}. I use the linear rectified path: given clean latent $Z_1$ and Gaussian noise $Z_0 \sim \mathcal{N}(0, I)$, the interpolated state is $Z_t = t\,Z_1 + (1-t)\,Z_0$, and a velocity network $v_\theta(Z_t, t)$ is trained against the velocity target $Z_1 - Z_0$. The training loss is
\begin{equation}
\cL_{\text{FM}} \;=\; \mathbb{E}_{t, Z_1, Z_0}\Bigl[\, \bigl\lVert v_\theta(Z_t, t) - (Z_1 - Z_0)\bigr\rVert_2^2 \,\Bigr],
\label{eq:flow_matching}
\end{equation}
with $t$ drawn uniformly on the closed unit interval. At inference, the velocity field is integrated by a forward Euler step from $Z_0$ to recover a sample of $Z_1$.

The backbone is a Diffusion Transformer (DiT) \citep{peebles2023dit, ma2024sit} with adaLN-Zero timestep conditioning. RoPE \citep{su2021roformer} is supported for the geometry path as an ablation against learned absolute positional embeddings.

\paragraph{Schema-aware Stage~2 design.} Because the seven Stage~1 variants split into a flat and a split schema, Stage~2 has to read the schema from the checkpoint and route correctly. There are four model classes, instantiated according to which Stage~1 was used.

\begin{itemize}
\item \emph{Flat schema, generation.} A single \texttt{SceneDiT} operates over the 512-token latent. Self-attention only (no cross-attention), adaLN-Zero on the timestep. This is essentially \texttt{LayoutDiT} widened from 16 to 512 tokens.
\item \emph{Flat schema, mean colour.} For colour-residual checkpoints, the per-scene mean colour is not part of the latent (it is read from the shape embedding in Stage~1). I train a small \texttt{DCHead} that models a conditional Gaussian $q(\bar{c} \mid Z)$ from a permutation-invariant pool of $Z$, so the missing DC component can be recovered at generation time.
\item \emph{Split schema, generation.} Two stages: a small \texttt{LayoutDiT} generates $z_s$ from noise, and a deeper \texttt{GeometryDiT512} generates $z_g$ conditioned on $z_s$ by cross-attention at every block. The frozen Stage~1 decoder then produces the Gaussians.
\item \emph{Completion.} For the split schema, \texttt{CompletionDiT} conditions on $z_s$ via cross-attention while masking a random fraction of $z_g$ tokens (30--80\% coverage) and denoising the masked positions. For the flat structured case (variant 5), an unconditional variant \texttt{CompletionDiTUncond} performs the same masked denoising with self-attention only.
\end{itemize}
Completion is only meaningful when token $k$ corresponds to a spatial region, which holds for variants 5, 6, and 7. For the global-flat variants 1--4 a token is a global mixture and masking it has no spatial meaning; I skip completion for those.

\section{Configuration Summary for the Seven Experiments}
\label{sec:method_configs}

All Stage~1 runs share the architecture defaults: transformer width $w = 384$, 12 attention heads, 6 encoder layers, 12 decoder layers, $T = 512$ tokens, $d = 32$ per token, 8-frequency Fourier features. Training uses AdamW \citep{loshchilov2019adamw} with $\beta_1 = 0.9$, $\beta_2 = 0.999$, weight decay $10^{-2}$, base learning rate $10^{-4}$, cosine decay with warm restarts at period $T_0 = 800$ epochs, and 300-step linear warmup. The KL weight is $\beta = 10^{-5}$, and training is in bf16 mixed precision on 2$\times$H100 GPUs (Snellius project EINF-16254). Batch size is 90 for the runs without local encoder (variants 1--4) and 64 for the runs with local encoder (variants 5--7). Validation runs every 10 epochs.

Three observations are worth making explicit. First, the only difference between V6 and V7 is the position scaffold, which gives a clean A/B test of what the anchor decomposition contributes once everything else (Hilbert order, local encoder, token-local decoder, disentanglement, per-Gaussian InfoNCE) is in place. Second, V1, V4, V5 share opacity-rank input order (no Hilbert reorder), so they are not strictly comparable to the reordered variants on the position component of the loss; they are nonetheless useful for isolating the decoder change (V1 vs.\ V4) and the encoder change (V4 vs.\ V5) under controlled conditions. Third, the parameter count difference between the flat decoder (about 777M) and the token-local decoder (about 1.6M) is roughly two orders of magnitude, so any comparison between V1--V3 and V4--V7 mixes a capacity change with an architectural change. I revisit both observations in Chapter~\ref{chap:discussion}.
