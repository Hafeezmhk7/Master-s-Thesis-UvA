\section{Hardware and Software Environment}
\label{app:env}

All experiments were run on Snellius (project EINF-16254) on a single node with $2 \times $ NVIDIA H100 80\,GB GPUs. The software stack is PyTorch 2.5 with CUDA 12.4, mixed-precision bf16, FlashAttention-2, and PyTorch Distributed (NCCL) for multi-GPU training. Each Stage~1 run takes approximately 36--48 hours wall-clock; the Stage~2 runs take an additional 12--24 hours per variant.

\section{300-Scene Configurations}
\label{app:300_configs}

The 300-scene runs used a fixed split of 180/60/60 train/val/test scenes from the chunked ScanNet subset of SceneSplat-7K, batch size 16, and 200 training epochs. Hyperparameters otherwise match the large-scale runs (Table~\ref{tab:app_hp}). The four disentanglement strategies (A, B1, B2, C) and the three InfoNCE flavours (pgNCE, PoolNCE, Pool+pgNCE) are toggled by command-line flags as described in the released codebase.

\section{Reproducibility Notes}
\label{app:repro}

\paragraph{Dataset locations.} Preprocessed scenes are under \texttt{/gpfs/work1/0/prjs2084/Hafeez\_thesis/Semantic-Scene-Completion/data/} with the following structure: \texttt{train/} (3{,}800 chunks), \texttt{interior\_gs/} (800 full scenes), \texttt{arkitscenes/}, \texttt{scannetpp/}, \texttt{val\_full/} (100 held-out full scenes), \texttt{val\_chunks/} (88 held-out chunks).

\paragraph{Seeds.} Each run uses a fixed seed of 42 for the PyTorch RNG and a fixed seed of 0 for the NumPy RNG used in the dataset shuffling. The Hilbert key computation is deterministic.

\paragraph{Logging.} Training logs (per-batch loss, per-epoch validation) are written to \texttt{logs/<JOB\_ID>.out}. Checkpoints are saved every 10 epochs to \texttt{checkpoints/<run\_name>/}.
