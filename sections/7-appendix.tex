\paragraph{Input feature construction.}
Each of the $N$ Gaussians contributes an 18-dimensional raw feature:
voxel-centre coordinates (3, FNV-hashed $40^3$ grid), voxel ID (1),
normalised position (3), colour (3), opacity (1), scale (3), quaternion (4).
A Fourier encoding of the voxel centre and the position (8 frequencies each,
$2\times3\times8=48$ dims each) plus the raw geometric parameters (8 dims)
give $110$ inputs per Gaussian, projected to $w=384$ by a linear layer.

\section{Dataset Details}
\label{sec:apx_data}

\paragraph{Training set.}
Main training: 3,800 interior chunks from ScanNet/Matterport3D/Replica,
preprocessed at $8{\times}8$ tile stride with $6{\times}6$ overlap.
Each chunk includes a \texttt{norm\_factor.npy} file encoding the global
room coordinate frame (centre + scale). Semantic labels (ScanNet-72) are
active for these chunks.

Additional full scenes (semantic labels disabled): 800 interior full scenes,
1,290 ARKitScenes, 856 ScanNet++ v2. Total training: $\approx$6,746 scenes.

\paragraph{Validation.}
Primary: 100 held-out full scenes from \texttt{interior\_gs/val/}
(per-scene normalisation).
Secondary: 88 held-out chunks (sorted after the first 3,800 training chunks;
disjoint by construction; global-frame normalisation via shared
\texttt{norm\_factor.npy}).

\paragraph{Data loading.}
\texttt{--no\_preload} is used due to the large training mix
($\approx$34\,GB per DDP rank at preload). The validation sets are always
preloaded (sampled once, frozen for the run).

\section{Diagram Making Guide}
\label{sec:apx_diagrams}

All architecture diagrams in this thesis have placeholder captions. We
recommend the following platforms for creating the final versions:

\paragraph{draw.io (diagrams.net).}
Free, web-based, at \url{https://app.diagrams.net/}. Supports multi-layer
diagrams, custom shapes, and PDF/PNG export. Suitable for all pipeline
and architecture diagrams (Figures~\ref{fig:pipeline}, \ref{fig:decoder},
\ref{fig:encoder}, \ref{fig:stage2}).

\emph{How to use:} Open a new diagram $\to$ drag rectangle shapes for
model blocks $\to$ use arrows for data flow $\to$ colour by function
(blue = encoder, orange = decoder/generator, green = latent, grey = data)
$\to$ File~$\to$~Export As~$\to$~PDF.

\paragraph{Matplotlib (Python).}
For quantitative plots (Figures~\ref{fig:hilbert}, \ref{fig:convergence},
\ref{fig:components}), use the scripts below. All required data is in
the Snellius log files.

\begin{verbatim}
# Extract convergence data from log files
grep "Val FULL SCENES" logs_ss1/can3tok_<JOB>.out \
  | awk '{print $4, $6}' > val_loss_r1.txt
# Then plot with matplotlib
import matplotlib.pyplot as plt
import numpy as np
data = np.loadtxt('val_loss_r1.txt')
plt.plot(data[:,0], data[:,1], label='R1')
plt.xlabel('Epoch'); plt.ylabel('Val L2 error')
plt.legend(); plt.savefig('convergence.pdf')
\end{verbatim}

\paragraph{SuperSplat.}
For qualitative reconstruction figures (Figures~\ref{fig:recon_300},
